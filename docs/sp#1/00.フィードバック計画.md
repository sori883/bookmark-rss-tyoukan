# Sprint #1 フィードバック計画

手動試験後のユーザーフィードバックを元にした改善計画。

## タイムゾーン方針

本システムは **JST (Asia/Tokyo, UTC+9)** を基準とする。

- **DB**: PostgreSQL の `timestamptz` で保存（内部的にはUTC、標準的な運用）
- **API レスポンス**: ISO 8601 文字列（`toISOString()`）で返却。フロントで `toLocaleDateString('ja-JP')` により JST 表示
- **通知時間**: `notification_hour` は JST の 0〜23 時
- **AI サービス**: `datetime.now(JST)` で現在時刻を取得し、通知対象をフィルタ
- **EventBridge Scheduler**: `Asia/Tokyo` タイムゾーンで毎時実行

## FB-1: 記事クリック時に既読にならない

### 現状の問題

`ArticleCard` コンポーネントでは記事タイトルが外部リンク（`target="_blank"`）として実装されており、クリック時に既読APIを呼んでいない。`api-client.ts` に `updateArticle` メソッドも存在しない。

バックエンドの `PATCH /articles/:id` エンドポイントは実装済み。

### 対応方針

- `api-client.ts` に `updateArticle(id, { is_read: true })` メソッド追加
- `use-articles.ts` に `useMarkArticleRead` フック追加（楽観的更新）
- `article-card.tsx` でリンククリック時に既読APIを呼び出す `onClick` ハンドラ追加

### 影響範囲

- `apps/web/src/lib/api-client.ts`
- `apps/web/src/hooks/use-articles.ts`
- `apps/web/src/components/articles/article-card.tsx`
- `apps/web/src/types/api.ts`（`UpdateArticleRequest` 型追加）

## FB-2: フィード登録後に記事一覧が更新されない

### 現状の問題

`useCreateFeed` の `onSuccess` で `['feeds']` のみ invalidate しており、`['articles']` を invalidate していない。バックエンドは初回フェッチ実装済みだが、フロントのキャッシュが古いまま。

### 対応方針

- `use-feeds.ts` の `useCreateFeed` の `onSuccess` に `queryClient.invalidateQueries({ queryKey: ['articles'] })` を追加

### 影響範囲

- `apps/web/src/hooks/use-feeds.ts`

---

## FB-3: フィード登録時のRSS自動検出

### 現状の問題

ユーザーはRSSフィードの直接URLを知っている必要がある。サイトURLからRSSを自動検出する機能がない。

### 対応方針

サーバー側でURLがRSSフィードでない場合に自動検出を試みる。以下の順で試行:

1. HTMLの `<link rel="alternate" type="application/rss+xml">` タグ検出
2. よくあるRSSパスの試行（`/feed`, `/rss`, `/atom.xml`, `/feed.xml`, `/rss.xml`, `/index.xml` 等）
3. 全て失敗 → エラーメッセージ返却

### 影響範囲

- `services/feed/src/lib/rss-discovery.ts`（新規）
- `services/feed/src/routes/feeds.ts`（`POST /feeds` の内部ロジック変更）
- `services/feed/src/services/feed-service.ts`（必要に応じて）

## FB-4: ブックマーク検索が動作しない

### 現状の問題

DBトリガーは `to_tsvector('japanese', ...)` で search_vector を生成しているが、検索クエリ側は `plainto_tsquery('english', ...)` を使用。言語設定の不一致で検索結果が返らない。

### 対応方針

- `bookmark-service.ts` の検索クエリで `'english'` → `'japanese'` に修正

### 影響範囲

- `services/feed/src/services/bookmark-service.ts`

## FB-6: Webhook URLの平文表示（セキュリティ）

### 現状の問題

`GET /settings` がWebhook URLをそのまま返し、フロントで平文表示している。ブラウザのDevToolsやネットワークタブから閲覧可能。

### 対応方針

- バックエンド `GET /settings`:
  - `webhook_url` の代わりに `webhook_url_registered: boolean` を返す
- バックエンド `PUT /settings`:
  - レスポンスでも `webhook_url_registered: boolean` を返す
- `GET /notification-targets`（サービスJWT専用）は変更なし（通知送信に必要）
- フロント:
  - 登録済み → 「登録済み」表示 + 変更ボタン
  - 未登録 or 変更時 → 入力フィールド表示

### 仕様変更

`04.API型定義.md` の `SettingsResponse` を更新:

```typescript
type SettingsResponse = {
  webhook_url_registered: boolean  // 変更: URLの代わりに登録状態
  webhook_type: "slack" | "discord" | null
}
```

### 影響範囲

- `docs/sp#1/04.API型定義.md`
- `services/feed/src/routes/settings.ts`
- `apps/web/src/components/settings/settings-form.tsx`
- `apps/web/src/types/api.ts`

## FB-8: 通知時間の設定変更

### 現状の問題

EventBridge Scheduler で毎朝9時固定。ユーザーごとに通知時間を変更できない。

### 対応方針

- **方式**: EventBridge を毎時実行に変更し、AIサービス側で各ユーザーの設定時刻と現在時刻(JST)を比較して通知対象をフィルタ
- **タイムゾーン**: JST (UTC+9) 基準
- **粒度**: 1時間単位（0〜23時）

### 仕様変更

`03.データモデル.md` の settings テーブルに `notification_hour` カラム追加:

| カラム | 型 | 説明 |
|---|---|---|
| notification_hour | integer | 通知時間 JST (0-23, デフォルト 9) |

`04.API型定義.md`:

```typescript
type SettingsResponse = {
  webhook_url_registered: boolean
  webhook_type: "slack" | "discord" | null
  notification_hour: number  // 追加: 0-23
}

type UpdateSettingsRequest = {
  webhook_url?: string
  webhook_type?: "slack" | "discord"
  notification_hour?: number  // 追加: 0-23
}
```

`GET /notification-targets` レスポンスに `notification_hour` 追加。

### 影響範囲

- `docs/sp#1/03.データモデル.md`
- `docs/sp#1/04.API型定義.md`
- `packages/db/src/schema.ts` + マイグレーション
