# AI共創基盤 - API型定義

サービス間のリクエスト/レスポンス型を定義する。
実装時は OpenAPI spec として `openapi/` 配下に出力し、各言語のクライアントを自動生成する。

## 共通型

```typescript
// ページネーション
type PaginationParams = {
  page?: number       // default: 1
  limit?: number      // default: 20, max: 100
}

type PaginatedResponse<T> = {
  data: T[]
  total: number
  page: number
  limit: number
}

// エラー
type ErrorResponse = {
  error: {
    code: string      // "NOT_FOUND", "UNAUTHORIZED", "VALIDATION_ERROR" 等
    message: string
  }
}
```

## auth-service

### POST /auth/service-token

サービス間認証用JWT発行。

```typescript
// Request
type ServiceTokenRequest = {
  client_id: string
  client_secret: string
}

// Response 200
type ServiceTokenResponse = {
  access_token: string
  token_type: "Bearer"
  expires_in: number    // 秒（例: 86400 = 24h）
}
```

### POST /auth/refresh

```typescript
// Request
type RefreshRequest = {
  refresh_token: string
}

// Response 200
type RefreshResponse = {
  access_token: string
  refresh_token: string
  expires_in: number
}
```

### GET /auth/me

```typescript
// Response 200
type AuthMeResponse = {
  id: string
  email: string
  name: string
}
```

### GET /auth/.well-known/jwks.json

```typescript
// Response 200（標準JWKS形式）
type JWKSResponse = {
  keys: {
    kty: string
    kid: string
    alg: string
    n: string
    e: string
    use: "sig"
  }[]
}
```

### RSS記事

#### POST /articles

```typescript
// Request
type CreateArticleRequest = {
  feed_id: string
  url: string
  title: string
  published_at: string    // ISO 8601
}

// Response 201
type ArticleResponse = {
  id: string
  user_id: string
  feed_id: string
  url: string
  title: string
  description: string       // 記事概要（RSSのdescription）
  is_read: boolean
  is_bookmarked: boolean   // ブックマーク済みフラグ
  published_at: string
  created_at: string
  updated_at: string
}
```

#### GET /articles

```typescript
// Query
type ListArticlesQuery = PaginationParams & {
  feed_id?: string
  is_read?: boolean
}

// Response 200
type ListArticlesResponse = PaginatedResponse<ArticleResponse>
```

```typescript
// Response 200
type GetArticleResponse = ArticleResponse
```

#### PATCH /articles/:id

```typescript
// Request

  is_read?: boolean
```

```typescript
// Response 204（No Content）
```

## bff（外部公開API）

bff の型は feed-service / notification-service と同一構造。
bff は下流サービスのレスポンスをそのまま返す（変換しない）ため、型定義は feed-service / notification-service を参照。

| bff エンドポイント | 委譲先 |
|---|---|
| GET /feeds | feed: GET /feeds |
| POST /feeds | feed: POST /feeds → POST /feeds/fetch |
| DELETE /feeds/:id | feed: DELETE /feeds/:id |
| POST /feeds/import-opml | feed: POST /feeds/import-opml |
| GET /articles | feed: GET /articles |
| GET /articles/:id | feed: GET /articles/:id → PATCH /articles/:id (既読) |
| GET /bookmarks | feed: GET /bookmarks |
| GET /bookmarks/:id | feed: GET /bookmarks/:id |
| POST /bookmarks | feed: POST /bookmarks |
| DELETE /bookmarks/:id | feed: DELETE /bookmarks/:id |
| GET /bookmarks/search | feed: GET /bookmarks/search |
| GET /notifications | notification: GET /notifications |
| PATCH /notifications/:id | notification: PATCH /notifications/:id |
| GET /settings | DB直接 |
| PUT /settings | DB直接 |

### GET /settings

```typescript
// Response 200
type SettingsResponse = {
  webhook_url_registered: boolean  // Webhook URLが登録済みか（URLは返さない）
  webhook_type: "slack" | "discord" | null
  notification_hour: number        // 通知時間 (0-23, デフォルト 9)
}
```

### PUT /settings

```typescript
// Request
type UpdateSettingsRequest = {
  webhook_url?: string             // 新規登録・変更時のみ送信
  webhook_type?: "slack" | "discord"
  notification_hour?: number       // 0-23
}

// Response 200
type UpdateSettingsResponse = SettingsResponse
```

### GET /settings/notification-targets（サービスJWT専用）

```typescript
// Response 200
type NotificationTargetsResponse = {
  data: {
    user_id: string
    webhook_url: string            // 通知送信に必要なため実URLを返す
    webhook_type: string
    notification_hour: number
  }[]
}
```
