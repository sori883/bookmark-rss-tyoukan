# AI共創基盤 - 要求定義

## 概要

RSSフィードの購読・AI要約通知と、Web記事のブックマーク・本文抽出を統合したWebアプリケーション。
保存した記事はDBに蓄積し、全文検索やMCP/CLI経由でAIに渡せるようにする。

## 対象ユーザ

- 少人数利用（不特定多数、登録制限なし）
- Google OAuthによる認証

## 機能要求

### F1. RSS管理

- F1-1. RSSフィードURLの登録・削除
- F1-2. 登録フィードの定期取得（30分間隔で自動取得）
- F1-3. 取得した記事一覧の表示（タイトル、日時、フィード名）
- F1-4. 記事の既読/未読管理
- F1-5. OPMLインポート（既存RSSリーダーからのフィード一括登録）

### F2. AI記事選定・通知

- F2-1. 定期ジョブ（毎朝9時等）でAIが新着記事から注目記事を選定し要約（AIモデルが興味のある記事を判断）
- F2-2. 選定した記事のURL + 要約を通知
  - Webhook通知（Slack / Discord）。Webhook URLはWebUIの設定画面で指定
- F2-3. 通知履歴はWebUI上で閲覧可能

### F3. ブックマーク

- F3-1. RSS記事のブックマーク（ワンクリックで追加）
- F3-2. ブックマーク登録時に記事本文を自動抽出（HTML→プレーンテキスト/Markdown）
- F3-3. ブックマーク一覧の表示・検索
- F3-4. ブックマークの削除
- F3-5. RSS以外のブックマーク登録（WebUIからURL手動入力で登録）

### F4. 記事蓄積・検索基盤

- F4-1. 抽出した記事本文をDBに保存（保持期間は無期限）
- F4-2. MCPサーバーとして記事検索・取得・ブックマーク操作を提供
- F4-3. CLIから記事検索・取得・ブックマーク操作が可能
- F4-4. キーワード検索（ブックマークのタイトル・本文を全文検索）

### F5. WebUI

- F5-1. ユーザ認証（Google OAuth）
- F5-2. RSSフィード管理画面（フィード登録/削除、OPMLインポート）
- F5-3. RSS記事一覧・閲覧画面（ブックマークボタン付き）
- F5-4. ブックマーク管理画面
- F5-5. ブックマーク本文の閲覧（抽出済みMarkdownの表示）
- F5-6. 設定画面（Webhook URL設定、通知設定）
- F5-7. レスポンシブ対応（PC/スマホ）
- F5-8. PWA対応（ホーム画面追加、オフライン閲覧）

### F6. ログ

- F6-1. 構造化ログ（JSON形式）
- F6-2. ログレベル管理（debug / info / warn / error）
- F6-3. 主要な操作・イベントのログ出力（RSS取得、要約生成、ブックマーク操作、認証、エラー等）
