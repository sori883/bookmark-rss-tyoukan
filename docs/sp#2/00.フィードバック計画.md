## FB-1: 絵文字アイコンを高品質なSVGアイコンに置換

### 現状の問題

- サイドバーのナビゲーションに絵文字（📰📡🔖🔔⚙️）を使用

| 📰 | `Newspaper` | 記事 |

| 🔔 | `Bell` | 通知 |

### 影響ファイル

## FB-4: ブックマーク検索エラー（CRITICAL）

### 現状の問題

エラー: `text search configuration "japanese" does not exist`

PostgreSQL 17 のデフォルトイメージには日本語テキスト検索設定が含まれていない。`custom-migration.sql` と `bookmark-service.ts` で `'japanese'` を指定しているが、このconfiguarionは別途インストールが必要。

### 対応方針

PostgreSQL に日本語テキスト検索設定を追加する方法:

1. **`pg_bigm` 拡張を使う** — 日本語2-gram検索。PostgreSQL標準外のため Docker イメージのカスタムが必要
2. **`simple` 設定にフォールバック** — 言語非依存のトークナイザー。日本語の形態素解析はできないが、部分一致検索として機能する
3. **`pgroonga` 拡張を使う** — 高機能な日本語全文検索。Docker イメージのカスタムが必要

**推奨: `simple` 設定を使用**
- Docker イメージのカスタム不要で即座に動作
- 日本語テキストでも空白区切りやN-gram的な部分マッチが可能
- 将来的に本格的な日本語検索が必要なら `pgroonga` に移行

### 変更内容

- `packages/db/src/custom-migration.sql`: `'japanese'` → `'simple'`
- `services/feed/src/services/bookmark-service.ts`: `'japanese'` → `'simple'`
- E2Eテスト: ブックマーク検索のE2Eテストを追加
- DB リセット + マイグレーション再実行が必要

### 影響ファイル

- `packages/db/src/custom-migration.sql`
- `services/feed/src/services/bookmark-service.ts`
- `apps/web/e2e/bookmarks.spec.ts` — 検索E2Eテスト追加

## FB-5: OPMLインポート後にフィードを即座に取得

### 現状の問題

`POST /feeds/import-opml` はフィードをDBに登録するだけで、初回フェッチを実行しない。個別の `POST /feeds` では初回フェッチが実装済みだが、OPMLインポートでは省略されている。

### 対応方針

- `feeds.ts` の `POST /feeds/import-opml` で、インポート後に各フィードの初回フェッチを実行
- フェッチは非同期で行い、失敗してもインポート自体は成功扱い（既存の `POST /feeds` と同じ方式）

### 影響ファイル

- `services/feed/src/routes/feeds.ts`

---

## FB-6: 削除ボタンの悪目立ち改善

### 現状の問題

- ブックマークカード・フィード一覧で `variant="danger"`（赤背景白文字）の削除ボタンが常時表示
- カード全体の中で削除ボタンが最も目立っており、メインコンテンツより注目を集めてしまう

### 対応方針

- 削除ボタンを `variant="ghost"` に変更し、lucide-react の `Trash2` アイコンのみ表示
- ホバー時に赤色に変化して削除であることを示す
- テキスト「削除」は非表示にし、`aria-label` でアクセシビリティ担保

### 影響ファイル

- `apps/web/src/components/bookmarks/bookmark-card.tsx`
- `apps/web/src/components/feeds/feed-list.tsx`
