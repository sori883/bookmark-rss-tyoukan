# シードデータとフレームワーク管理データの境界

## 原則

テストデータのシード投入時は、**アプリケーション管理データ**と**フレームワーク管理データ**を区別する。

- **アプリケーション管理データ**: アプリケーションコードが直接CRUD するデータ。シードで自由に投入してよい。
- **フレームワーク管理データ**: フレームワークやライブラリが内部的に生成・管理するデータ。暗号化・シリアライズ等の独自フォーマットを持つ場合がある。

## リスク

フレームワーク管理データをシードで投入すると、フォーマットの不整合（暗号化方式、バージョン固有の内部構造等）により実行時エラーを引き起こす。

## ルール

- フレームワークが自動生成するデータはシードに含めず、フレームワーク自身に初期化させる
- シード対象はアプリケーション管理データに限定する
- 判断に迷う場合は、フレームワークのドキュメントで該当テーブル/データの管理主体を確認する
